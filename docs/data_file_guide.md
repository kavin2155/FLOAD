# 데이터 파일 구조 설명

이 문서는 FLOAD 데이터 폴더에서 원천데이터와 라벨링데이터를 구분하기 위한 안내서입니다.

처음 데이터를 보는 팀원은 아래 세 가지만 먼저 이해하면 됩니다.

```text
원천데이터 = 모델이 직접 보는 이미지/영상 파일
라벨링데이터 = 원천데이터에 대한 정답/설명 파일
Training = 모델 학습용 데이터
Validation = 모델 검증용 데이터
```

## 기본 약어

데이터 파일명에는 `TS`, `TL`, `VS`, `VL` 같은 약어가 붙습니다.

```text
TS = Training Source
TL = Training Label
VS = Validation Source
VL = Validation Label
```

즉, 이름만 보면 아래처럼 이해하면 됩니다.

```text
TS.zip = 학습용 원천데이터
TL.zip = 학습용 라벨링데이터
VS.zip = 검증용 원천데이터
VL.zip = 검증용 라벨링데이터
```

## 원천데이터와 라벨링데이터

`원천데이터`는 모델 입력으로 사용할 이미지나 영상입니다.

예를 들면 침수 위험 이미지, CCTV 영상, 도로 이미지 같은 파일이 여기에 해당합니다.
모델은 이 파일을 보고 침수인지 아닌지 판단하도록 학습합니다.

`라벨링데이터`는 원천데이터의 정답 정보입니다.

예를 들면 특정 이미지가 어떤 침수 단계인지, 어떤 지역인지, 어떤 조건에서 만들어진 데이터인지 같은 정보가 들어갑니다.
학습할 때는 원천데이터와 라벨링데이터를 짝지어 사용합니다.

## 부산시 침수위험 복합 데이터

현재 부산 데이터는 Drive의 아래 위치에 있습니다.

```text
FLOAD_DATA/
  raw/
    135_busan_flood/
      original/
        135.부산시_침수위험_복합_데이터/
          3.개방데이터/
            1.데이터/
```

그 아래 구조는 대략 아래와 같습니다.

```text
1.데이터/
  Training/
    01.원천데이터/
      TS.zip.part*
    02.라벨링데이터/
      TL.zip.part*
  Validation/
    01.원천데이터/
      VS.zip.part*
    02.라벨링데이터/
      VL.zip.part*
  Other/
    Other.zip.part*
```

### Training 원천데이터

```text
Training/01.원천데이터/TS.zip.part*
```

학습용 이미지 원본입니다.

현재 먼저 압축을 풀고 있는 데이터가 이 파일입니다.
압축을 풀면 아래와 같은 폴더가 나옵니다.

```text
침수위험 수치모델 이미지 데이터/
침수위험 지역 라벨링 데이터/
```

이 데이터는 부산 침수 탐지 모델을 학습할 때 가장 먼저 확인할 이미지 데이터입니다.

### Training 라벨링데이터

```text
Training/02.라벨링데이터/TL.zip.part*
```

학습용 원천데이터에 대응되는 정답/메타데이터입니다.

이미지만 있으면 모델이 무엇을 정답으로 배워야 하는지 알 수 없기 때문에, 실제 학습에는 `TS`와 `TL`을 같이 확인해야 합니다.

### Validation 원천데이터

```text
Validation/01.원천데이터/VS.zip.part*
```

검증용 이미지 원본입니다.

모델을 학습한 뒤 성능을 확인할 때 사용합니다.
처음부터 전부 풀 필요는 없고, 학습 데이터 구조를 먼저 잡은 다음 확인해도 됩니다.

### Validation 라벨링데이터

```text
Validation/02.라벨링데이터/VL.zip.part*
```

검증용 원천데이터에 대응되는 정답/메타데이터입니다.

검증 성능을 계산할 때 `VS`와 짝지어 사용합니다.

## 07 CCTV 데이터

07 CCTV 데이터는 Drive의 아래 위치에 있습니다.

```text
FLOAD_DATA/
  raw/
    07_cctv/
      original/
        07.지능형_관제_서비스_CCTV_영상_데이터/
          3.개방데이터/
            1.데이터/
```

이 데이터도 같은 규칙을 따릅니다.

```text
Training/01.원천데이터/TS_*.zip.part*
Training/02.라벨링데이터/TL_*.zip.part*
Validation/01.원천데이터/VS_*.zip.part*
Validation/02.라벨링데이터/VL_*.zip.part*
```

파일명에는 침수 단계와 도로 유형이 같이 들어갑니다.

```text
침수_0단계 = 침수 없음 또는 매우 낮은 단계
침수_1단계 = 침수 초기/낮은 단계
침수_2단계 = 침수 심화 단계

대로 = 큰 도로
골목길 = 좁은 도로
교차로 = 교차로
기타 = 기타 유형
```

예시는 아래처럼 읽으면 됩니다.

```text
TS_침수_0단계_대로.zip.part*
= 학습용 원천데이터, 침수 0단계, 대로 유형

TL_침수_0단계_대로.zip.part*
= 위 원천데이터에 대응되는 학습용 라벨링데이터

VS_침수_1단계_교차로.zip.part*
= 검증용 원천데이터, 침수 1단계, 교차로 유형

VL_침수_1단계_교차로.zip.part*
= 위 검증용 원천데이터에 대응되는 검증용 라벨링데이터
```

## 압축파일을 다 풀어야 하는가

처음부터 모든 압축파일을 다 풀 필요는 없습니다.

파일 수가 많고 용량이 크기 때문에 아래 순서로 푸는 것을 권장합니다.

```text
1. 부산 Training 원천데이터 TS
2. 부산 Training 라벨링데이터 TL
3. 필요한 경우 부산 Validation 원천데이터 VS
4. 필요한 경우 부산 Validation 라벨링데이터 VL
5. CCTV 데이터는 필요한 침수 단계/도로 유형부터 선택적으로 압축 해제
```

모델 학습을 시작하려면 최소한 아래 두 개가 먼저 필요합니다.

```text
Training 원천데이터
Training 라벨링데이터
```

## 팀원이 데이터를 볼 때의 기준

팀원이 Drive에서 데이터를 볼 때는 아래 순서로 확인하면 됩니다.

```text
1. 데이터가 부산 데이터인지, 07 CCTV 데이터인지 확인
2. Training인지 Validation인지 확인
3. 원천데이터인지 라벨링데이터인지 확인
4. TS/TL/VS/VL 약어 확인
5. 필요한 압축파일만 로컬에서 압축 해제
```

원본 압축파일은 보관용입니다.
학습에 바로 쓰기 좋게 정리한 파일은 나중에 `processed/` 아래에 따로 정리합니다.
